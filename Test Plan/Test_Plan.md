# Test Plan – E-commerce Website

## 1. Project Name

E-commerce Website Testing

## 2. Objective

The objective of this test plan is to verify that the e-commerce website functions correctly, securely, and provides a smooth shopping experience to users.

## 3. Scope

### In Scope

* User Registration
* User Login
* Forgot Password
* Product Search
* Product Details
* Add to Cart
* Update Cart
* Remove from Cart
* Checkout
* Logout
* User Profile

### Out of Scope

* Payment Gateway Integration
* Performance Testing
* Security Testing
* Mobile Application Testing

## 4. Testing Types

* Functional Testing
* UI Testing
* Smoke Testing
* Regression Testing
* Database Validation (SQL)
* API Testing (Basic)

## 5. Test Environment

* Operating System: Windows 11
* Browser: Google Chrome
* IDE: PyCharm
* Database: MySQL (or Sample Database)
* API Tool: Postman
* Version Control: Git & GitHub

## 6. Entry Criteria

Testing will begin when:

* Application is deployed.
* Test environment is available.
* Requirements are understood.

## 7. Exit Criteria

Testing will be completed when:

* All planned test cases are executed.
* Critical and High severity defects are resolved.
* Test summary report is prepared.

## 8. Deliverables

* Test Plan
* Test Scenarios
* Test Cases
* Bug Reports
* SQL Queries
* API Test Collection
* Test Summary

## 9. Assumptions

* Test data is available.
* Stable internet connection.
* Application is accessible.

## 10. Risks

* Requirement changes.
* Environment issues.
* Limited testing time.

## 11. Tools Used

* PyCharm
* Git
* GitHub
* SQL
* Postman
* Selenium (Learning)
